# PRD: dotisan UI Overhaul — Candy-Sweet, Cute-as-Hell Terminal Experience

## Executive Summary

Replace the current `pterm`-based terminal UI with a richer, more expressive stack: `briandowns/spinner` for spinners, `aquasecurity/table` for tables with text wrapping and colored borders, and `Digital-Shane/treeview` for colorized tree output. Introduce a centralized color system so every visual element stays consistent and themeable from a single config point. Remove the progress bar in favor of spinners. Rewrite all user-facing copy in a "dark-cute" voice.

---

## Goals

1. **Replace pterm** as the primary UI rendering library (keep it only if needed for edge cases during transition)
2. **Introduce `briandowns/spinner`** — use spinner #28 (`.oO°Oo.`) as the default, replace all progress bar usage
3. **Introduce `aquasecurity/table`** — text wrapping to fit any terminal width, customizable border/line colors, color-coded rows by status
4. **Introduce `Digital-Shane/treeview`** — custom icons + per-item custom colors for diff/tree output
5. **Centralized color system** — global `ColorPalette` struct with getters/setters, define named colors: `Main`, `Text`, `Synced`, `Drift`, `Add`, `Remove`, `Warn`, `Info`, `Dim`, `Success`
6. **Color-coded table rows** — each row's status maps to a palette color (Add→green, Remove→red, Drift→yellow, Sync→cyan, etc.)
7. **"Dark-cute" copy** — rewrite all user-facing strings (headers, statuses, success messages, errors) to be engaging, cute, and slightly edgy
8. **No background colors anywhere** — foreground text, borders, and spinner colors only
9. **Non-JSON output mode only** — human-facing output uses the new styled components; JSON mode unaffected
10. **Keep the "no changes" rainbow kudos box** — preserve existing `RenderNoChanges()` behavior (random kudos, rainbow title, purple border)

---

## Current State (Baseline)

| Area | Current Library | Limitation |
|---|---|---|
| Tables | `pterm.TablePrinter` | No text wrapping; narrow terminals break layout; no per-cell color; border style locked |
| Spinners/Progress | `pterm.SpinnerPrinter` + `pterm.ProgressbarPrinter` | Spinner styles limited; progress bar is clunky for our use case |
| Trees | `pterm.TreeFromLeveledList` | No per-item color; no custom icons per node type |
| Colors | `pkg/style/styles.go` — ad-hoc `pterm.Color` constants | No centralized palette; no getters/setters; changes require touching many files |
| Copy | Mixed tone, some cute (nochanges), rest is plain/neutral | Inconsistent voice |

## Requirements

### FR-1: Replace Progress Bar with Spinner
- **Remove** `pkg/style/progress.go` (NewApplyProgress, CompleteItem)
- **Use** `briandowns/spinner` for all long-running operations
- Default spinner: type **28** (`.oO°Oo.`)
- Spinner color: `ColorPalette.Main`
- Spinner text: dark-cute messages (e.g., `"✨ Syncing your dots..."`, `"🔮 Applying magic..."`)

### FR-2: New Table Rendering (`aquasecurity/table`)
- **Replace** `pkg/ui/table.go` and `pkg/ui/resources.go` pterm table usage
- Enable **text wrapping** so tables fit any terminal width automatically
- **Customizable border colors** via `ColorPalette` (border = `Main`, header text = `Main`)
- **Color-coded rows**: each `ResourceRow.Status` maps to a color:
  - `add` → `ColorPalette.Add` (green-ish)
  - `remove` → `ColorPalette.Remove` (red-ish)
  - `update/modify` → `ColorPalette.Warn` (yellow-ish)
  - `drift` → `ColorPalette.Drift` (orange-ish)
  - `sync/in_sync` → `ColorPalette.Synced` (cyan-ish)
- No background colors on any cell or row

### FR-3: New Tree Rendering (`Digital-Shane/treeview`)
- **Replace** `pkg/diff/tree.go` pterm tree usage
- Custom icons per node type (reuse existing `IconAdd`, `IconRemove`, `IconEdit`, etc. from `pkg/style/styles.go`)
- **Per-item color** based on diff state using `ColorPalette`
- Custom connector/branch colors via `ColorPalette.Dim`

### FR-4: Centralized Color Palette
Create `pkg/style/palette.go`:

```go
type ColorPalette struct {
    Main    Color  // primary accent (headers, borders)
    Text    Color  // default body text
    Synced  Color  // in-sync / success state
    Drift   Color  // drift detected
    Add     Color  // additions
    Remove  Color  // removals
    Warn    Color  // warnings, modifications
    Info    Color  // informational
    Dim     Color  // muted/deemphasized
    Success Color  // success highlights
}

func DefaultPalette() ColorPalette { ... }
func (p *ColorPalette) SetColor(name string, c Color) // setter
func (p ColorPalette) GetColor(name string) Color       // getter
```

- All UI packages (`ui/`, `diff/`, `style/`) reference `palette.ColorPalette` instead of hard-coded `pterm.Color` values
- Changing `palette.DefaultPalette()` immediately affects entire app

### FR-5: Dark-Cute Copy Rewrite
Rewrite all user-facing text strings. Examples:

| Before | After (dark-cute) |
|---|---|
| `"Planning resources..."` | `"✨ Sniffing out your dots..."` |
| `"No changes detected"` | `"💅 Flawless. Zero. Nada. Touch grass."` |
| `"Error: failed to apply"` | `"💀 Oof. Things broke. Here's why:"` |
| `"Sync complete"` | `"🎀 All synced up, you absolute legend."` |
| `"Drift detected in X"` | `"👀 Caught X drifting like a rebellious cloud..."` |

- Apply to: spinner texts, table headers, section headers, success/error boxes, plan summaries
- Tone: cute, slightly edgy, engaging — never corporate

### FR-6: Preserve "No Changes" Box
- Keep `pkg/ui/nochanges.go` `RenderNoChanges()` intact
- Rainbow per-character title + dark purple border + white body — already perfect
- No changes needed to this component

### FR-7: No Background Colors
- Explicit constraint: nowhere in the app shall background colors be used
- Only manipulate: foreground text color, border/frame color, spinner character color
- All new components must respect this

### FR-8: Non-JSON Output Mode
- All changes apply to human-readable terminal output only
- JSON output mode (`--output json`) remains untouched, no styling applied

---

## Out of Scope

- Background color support (explicitly rejected)
- Interactive/TUI mode (not needed)
- Theme switching at runtime (palette is set at startup)
- Markdown or HTML export
