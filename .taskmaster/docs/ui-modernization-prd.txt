================================================================================
UI MODERNIZATION PRD - dotisan CLI Enhancement
================================================================================

## 1. Overview

**Project Name:** dotisan UI Modernization
**Status:** Proposed
**Priority:** High
**Target:** 2026 Quality CLI Experience

### Executive Summary

The dotisan CLI currently has a fragmented user experience - while the `plan`
command uses beautiful Charmbracelet lipgloss styling with pastel colors and
icons, other commands like `state list`, `state import`, `apply`, and `init`
use plain text output. This creates an inconsistent, dated feel that doesn't
match the 2026 vision.

This PRD proposes a comprehensive UI modernization to bring all commands to
the same high standard, creating a cohesive and delightful user experience.

### Current State Analysis

**What's Working Well:**
- `plan` command uses beautiful pastel palette (mint green, salmon, cream yellow, peach)
- Nice icons: ✚ ✖ ✎ ⚠ ✓ consistently used in plan
- `PlanFormatter` provides structured styling system
- lipgloss imported throughout codebase (121 references)

**What's Missing:**
- Only `plan` uses rich styling
- Success/error messages are plain text
- Tables use basic formatting
- No visual hierarchy or consistent branding
- Commands feel disconnected from each other

================================================================================
## 2. Problem Statement

### User Pain Points

1. **Inconsistent Experience**: Running different commands feels like using
   different apps - `plan` is beautiful, `state list` is plain

2. **Dated Feel**: Plain text output makes CLI feel like it's from 2001,
   not 2026

3. **Poor Readability**: No visual hierarchy makes it hard to scan output
   and find important information

4. **No Engagement**: Commands don't provide feedback, progress, or
   delightful micro-interactions

5. **No Emotional Design**: The CLI is purely functional, not inspiring

### Business Impact

- Users feel the project is "dead" or abandoned
- Reduced motivation to use dotisan daily
- Poor impression for new users evaluating the tool
- Missed opportunity for delighting users

================================================================================
## 3. Proposed Solution

### Design Principles

1. **Consistency First**: Every command uses the same style system
2. **Progressive Enhancement**: Basic colored output → rich UI → interactive
3. **Respect User Intelligence**: Don't over-animate, keep it tasteful
4. **Performance Matters**: Styling shouldn't slow down commands
5. **Accessibility**: Ensure colors work in common terminal themes

## 4. Technical Specification

### 4.1 Shared Style System

**New File:** `pkg/style/styles.go`

```go
package style

import "github.com/charmbracelet/lipgloss"

// Core colors
const (
    Green  = "114"
    Red    = "174"
    Yellow = "222"
    Orange = "216"
    Gray   = "240"
)

// Icon constants
const (
    IconAdd     = "✚"
    IconRemove = "✖"
    IconEdit   = "✎"
    IconWarn   = "⚠"
    IconOK     = "✓"
    IconInfo   = "ℹ"
)

// Text Styles
var (
    Success = NewStyle().Foreground(Green)
    Error   = NewStyle().Foreground(Red)
    Warning = NewStyle().Foreground(Orange)
    Info    = NewStyle().Foreground(Yellow)
    Dim     = NewStyle().Foreground(Gray)
)

// Icon Styles
var (
    IconSuccess = Success.Render(IconOK)
    IconError   = Error.Render(IconRemove)
    IconWarning = Warning.Render(IconWarn)
    IconInfo    = Info.Render(IconInfo)
)

// Border Styles
var (
    SuccessBox = NewStyle().
        Border(lipgloss.RoundedBorder()).
        BorderForeground(Green).
        Padding(1, 2)

    ErrorBox = NewStyle().
        Border(lipgloss.RoundedBorder()).
        BorderForeground(Red).
        Padding(1, 2)
)

// Table Styles
var (
    TableHeader = NewStyle().Bold(true).Underline(true)
    TableRow    = NewStyle()
    TableCell   = NewStyle()
)
```

## 7. Non-Functional Requirements

### Performance
- Style rendering adds <5ms overhead per command
- No degradation in command execution time

### Compatibility
- Works in standard terminal (80x24 minimum)
- Colors degrade gracefully in limited terminals
- Supports dark and light themes

### Accessibility
- Color is not the only indicator (icons also used)
- High contrast option available
- Works with screen readers (structure preserved)

### Maintainability
- Single source of truth for styles
- Easy to update colors globally
- Well-documented style system
